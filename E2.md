# Entregable 2: Cronograma Estimado de Ejecución

## Duración Total: 6 Semanas

---

## 1. Resumen del Cronograma

| Semana       | Fase                | Actividades Principales                                           |
| ------------ | ------------------- | ----------------------------------------------------------------- |
| **Semana 1** | Preparación         | Configuración del entorno de pruebas y diseño de casos detallados |
| **Semana 2** | Ejecución Funcional | Pruebas Funcionales (Módulos individuales: Ventas, RRHH)          |
| **Semana 3** | Ejecución Funcional | Pruebas Funcionales (Marketing, Postventa) y Pruebas de Roles     |
| **Semana 4** | Integración         | Pruebas de Integración de Datos (Flujos de extremo a extremo)     |
| **Semana 5** | Estabilización      | Pruebas de Regresión y Corrección de Defectos (Bugs encontrados)  |
| **Semana 6** | Cierre              | Cierre de Pruebas, Informe Final y Firma de Aceptación (Sign-off) |

---

## 2. Detalle de Horas Estimadas por Fase

### Equipo de 2 QAs

| Fase                                          | Semanas     | Horas Totales | Actividades                                                                                                                |
| --------------------------------------------- | ----------- | ------------- | -------------------------------------------------------------------------------------------------------------------------- |
| **Fase 1: Preparación**                       | Semana 1    | 40 horas      | Creación de datos base (empleados, productos, leads), configuración de entorno de QA, diseño de casos de prueba detallados |
| **Fase 2: Ejecución Funcional e Integración** | Semanas 2-4 | 120 horas     | Pruebas manuales y automatización de flujos críticos (Ventas, RRHH, Marketing, Postventa, Roles)                           |
| **Fase 3: Estabilización y Cierre**           | Semanas 5-6 | 80 horas      | Re-testeo de errores corregidos por los desarrolladores, pruebas de regresión, generación de informes                      |

**Total estimado: 240 horas**

## 4. Deliverables por Semana

### Semana 1: Preparación

- [ ] Entorno de QA configurado
- [ ] Datos de prueba creados (empleados, productos, leads, clientes)
- [ ] Casos de prueba detallados documentados
- [ ] Plan de pruebas finalizado

### Semana 2: Pruebas Funcionales (Ventas, RRHH)

- [ ] CP-02: Prueba de generación de facturas
- [ ] CP-03: Prueba de procesamiento de nómina
- [ ] CP-06: Prueba de gestión de stock
- [ ] CP-10: Prueba de rechazo por stock insuficiente

### Semana 3: Pruebas Funcionales (Marketing, Postventa, Seguridad)

- [ ] CP-01: Prueba de conversión de Lead
- [ ] CP-04: Prueba de creación de ticket de soporte
- [ ] CP-05: Prueba de restricción de acceso por rol
- [ ] CP-07: Prueba de validación de estado de Lead

### Semana 4: Pruebas de Integración

- [ ] E2E-01: Flujo completo Lead → Cliente → Venta → Soporte
- [ ] Pruebas de integración de datos entre módulos
- [ ] Verificación de traspasos entre Marketing → Ventas → Postventa

### Semana 5: Pruebas de Regresión

- [ ] Ejecución de suite de regresión
- [ ] Re-testeo de bugs corregidos
- [ ] Validación de correcciones

### Semana 6: Cierre

- [ ] Informe final de pruebas
- [ ] Resumen de defectos encontrados/cerrados
- [ ] Firma de aceptación (Sign-off)
- [ ] Entrega de documentación

## 5. Roles y Responsabilidades

| Rol                  | Responsabilidad                                                          |
| -------------------- | ------------------------------------------------------------------------ |
| **QA Lead**          | Coordinación general, planificación, reporte a gerencia                  |
| **QA Engineer 1**    | Ejecución de pruebas funcionales (Ventas, RRHH, Marketing)               |
| **QA Engineer 2**    | Ejecución de pruebas funcionales (Postventa, Seguridad) y automatización |
| **Desarrolladores**  | Corrección de bugs reportados                                            |
| **Business Analyst** | Validación de reglas de negocio y criterios de aceptación                |

## 7. Métricas de Seguimiento

| Métrica                        | Descripción                              | Meta  |
| ------------------------------ | ---------------------------------------- | ----- |
| **% Casos Ejecutados**         | Casos ejecutados / Total de casos        | > 90% |
| **% Casos Aprobados**          | Casos aprobados / Total ejecutados       | > 95% |
| **Defectos Críticos Abiertos** | Defectos con prioridad Alta sin resolver | 0     |
| **Defectos Bloqueantes**       | Defectos que bloquean la liberación      | 0     |
| **Cobertura de Pruebas**       | Requisitos cubiertos por casos de prueba | > 80% |
